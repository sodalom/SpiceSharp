Syntax
public abstract class Entity : ICloneable, ICloneable<Entity>
Remarks

Entities should not contain references to other entities, but only their name identifiers. In the method BindBehavior(IBehavior, Simulation) the entity should try to find the necessary behaviors and parameters generated by other entities and pass them via a BindingContext.

CreateBehaviors(Type[], Simulation, EntityCollection)

Creates behaviors of the specified types. The type order is important.

Declaration
public virtual void CreateBehaviors(Type[] types, Simulation simulation, EntityCollection entities)
Parameters
Type Name Description
System.Type[] types

The types of behaviors that the simulation wants, in the order that they will be called.
Simulation simulation

The simulation requesting the behaviors.
EntityCollection entities

The entities being processed, used by the entity to find linked entities.
Remarks

The order typically indicates hierarchy. The entity will create the behaviors in reverse order, allowing the most specific child class to be used that is necessary. For example, the OP simulation needs ITemperatureBehavior and an IBiasingBehavior. The entity will first look for behaviors of type IBiasingBehavior, and then for the behaviors of type ITemperatureBehavior. However, if the behavior that was created for IBiasingBehavior also implements ITemperatureBehavior, then then entity will not create a new instance of the behavior.
